# Amethyst Post Bot

Telegram-бот для публичного создания постов в формате Telegram Rich Markdown. Пользователь отправляет Markdown текстом или `.md` файлом, бот проверяет подписку на канал, изменяет пост под новый формат и отправляет готовый rich-post.

Бот работает только в личных сообщениях с пользователем. Он не публикует посты в канал, чат или группу автоматически; канал используется только для проверки подписки.

## Возможности

- Принимает Markdown сообщением или UTF-8 файлом `.md`.
- Перед использованием требует подписку на канал `@AmethystSMPChannel`.
- Отправляет результат только пользователю в личные сообщения.
- Работает через SOCKS5-прокси, если указан `TELEGRAM_PROXY_URL`.
- Поддерживает удобные блоки `details`, `collage` и `slideshow`.
- Проверяет, что ссылки на медиа начинаются с `http://` или `https://`.
- Отправляет пример по команде `/example` и через кнопку “Показать пример”.

## Требования

- Python 3.10 или новее.
- Токен Telegram-бота от BotFather.
- Bot API endpoint с поддержкой `sendRichMessage`.
- Бот должен быть администратором в канале, чтобы проверять подписку через `getChatMember`.

## Быстрый старт

1. Установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

2. Скопируйте шаблон окружения:

   ```bash
   cp .env.example .env
   ```

3. Заполните `.env`:

   ```env
   TELEGRAM_BOT_TOKEN=123456789:your_real_bot_token
   TELEGRAM_PROXY_URL=socks5h://user:password@host:port
   REQUIRED_CHANNEL=@AmethystSMPChannel
   REQUIRED_CHANNEL_URL=https://t.me/AmethystSMPChannel
   CHAT_URL=https://t.me/amethystSMP
   ```

4. Запустите бота:

   ```bash
   python main.py
   ```

5. Откройте бота в Telegram и отправьте `/start`.

## Настройки

Реальные секреты должны лежать только в `.env`. Этот файл игнорируется Git.

| Переменная | Обязательная | Описание |
| --- | --- | --- |
| `TELEGRAM_BOT_TOKEN` | Да | Токен Telegram-бота. |
| `TELEGRAM_PROXY_URL` | Нет | SOCKS5/SOCKS5H прокси для сервера в РФ. |
| `REQUIRED_CHANNEL` | Да | Канал, на который пользователь должен быть подписан. |
| `REQUIRED_CHANNEL_URL` | Нет | Публичная ссылка на канал для кнопки. |
| `CHAT_URL` | Нет | Публичная ссылка на чат для дополнительной кнопки. |
| `BOT_ENV_FILE` | Нет | Путь к другому env-файлу, если нужен нестандартный запуск. |

Переменные окружения имеют приоритет над значениями из `.env`.

## Команды

- `/start` - показывает приветствие, кнопки канала, чата и пример.
- `/help` - показывает краткую справку по формату постов.
- `/example` - отправляет исходник `sample_post.md`, а затем готовый rich-post.

Если пользователь не подписан на канал, бот покажет кнопки “Подписаться на канал”, “Открыть чат” и “Проверить подписку”. Создание постов будет заблокировано до подтверждения подписки на канал.

В группах, каналах и чатах бот не отвечает на сообщения, чтобы случайно не публиковать пользовательские посты вне личного диалога.

## Минигайд по постам

Отправьте Markdown напрямую сообщением или загрузите UTF-8 файл `.md`.

Обычный раскрывающийся блок:

```md
:::details Вопрос
Скрытый ответ.
:::
```

Блок, открытый по умолчанию:

```md
:::details open Вопрос
Этот текст сразу раскрыт.
:::
```

Коллаж:

```md
:::collage
![](https://example.com/photo-1.jpg "Первое фото")
![](https://example.com/photo-2.jpg "Второе фото")
:::
```

Слайд-шоу:

```md
:::slideshow
![](https://example.com/slide-1.jpg)
![](https://example.com/slide-2.jpg)
:::
```

Полный пример находится в [sample_post.md](sample_post.md). Его же бот отправляет по команде `/example`.

## Проверка перед релизом

- Убедитесь, что реальный `TELEGRAM_PROXY_URL` не попал в README, `.env.example` или код.
- Проверьте синтаксис:

  ```bash
  python -m py_compile main.py
  ```

- В Telegram проверьте неподписанного пользователя, кнопку “Проверить подписку”, `/example`, текстовый пост и загрузку `.md` файла.
